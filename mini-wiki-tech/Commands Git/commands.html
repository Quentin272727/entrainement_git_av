Commandes Git essentielles

Voici une liste de commandes Git courantes avec une brève description pour chaque commande. Ce guide rapide aide à effectuer les opérations Git de base.

Commande / Description
Commande Description
git init Initialise un nouveau dépôt Git local dans le répertoire courant.
git clone <url> Copie un dépôt distant (clone) sur votre machine locale.
git status Affiche les fichiers modifiés, ajoutés ou non suivis dans le dépôt.
git add <fichier> Ajoute les changements d'un fichier (ou git add . pour tous) à l'index en vue d'un commit.
git commit -m "message" Enregistre dans l'historique les modifications indexées avec un message descriptif.
git push [origin] [branche] Envoie les commits locaux vers le dépôt distant (par défaut origin).
git pull Récupère les changements distants et fusionne avec la branche locale courante.
git fetch Télécharge les objets et refs depuis le dépôt distant sans fusionner automatiquement.
git branch Liste les branches locales; git branch <nom> crée une nouvelle branche.
git checkout <branche|commit> Bascule vers une autre branche ou restore l'état à un commit spécifique.
git merge <branche> Fusionne la branche spécifiée dans la branche courante.
git rebase <branche> Rejoue vos commits sur une autre base (utile pour un historique linéaire).
git log Affiche l'historique des commits; utile avec --oneline ou --graph.
git reset --soft|--hard <commit> Réinitialise la HEAD à un commit donné: --soft garde l'index, --hard efface les changements locaux.

Astuce rapide : combinez git status, git add et git commit pour enregistrer vos modifications locales, puis git push pour les transmettre au dépôt distant.
